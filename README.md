## Project Overview

A supervised ML pipeline for credit card fraud detection using the Kaggle Credit Card
Fraud Detection dataset. The project studies class-imbalance handling (class weighting,
SMOTE, random under-sampling), dynamic decision threshold tuning, and cost-sensitive
evaluation under asymmetric FP/FN business costs.

**Problem type:** Binary classification (fraud = 1, legitimate = 0)  
**Dataset:** 284,807 transactions, 30 features (V1–V28 PCA + Time + Amount), 0.17% fraud prevalence

## Results Summary

Best model: **Random Forest (balanced class weights)**, threshold = 0.192

| Metric | Validation | Test |
|--------|-----------|------|
| PR-AUC | 0.8637 | 0.8116 |
| ROC-AUC | 0.9514 | 0.9376 |
| F1-score | 0.8767 | 0.8356 |
| Recall | 86.5% | 82.4% |
| Precision | 88.9% | 84.7% |
| Business Cost (€500 FN, €10 FP) | — | €6,610 |

All three models (LR, RF, GBT) and all three imbalance strategies evaluated and compared.

## Setup

**1. Extract the dataset**
```bash
unzip creditcard.csv.zip -d data/
```

**2. Create virtual environment and install dependencies**
```bash
python3 -m venv venv
source venv/bin/activate        # Windows: venv\Scripts\activate
pip install -r requirements.txt
```

**3. Launch Jupyter and run the final notebook**
```bash
jupyter notebook
# Open notebooks/fraud_detection_final.ipynb
# Kernel > Restart & Run All
```

Alternatively, execute headlessly:
```bash
jupyter nbconvert --to notebook --execute --inplace notebooks/fraud_detection_final.ipynb
```
